## Terms

| Term | Definition | Source |
|---|---|---|
| **Workspace** | A top-level tab in the sidebar; owns a shell, a working directory, an immutable split tree of panes, and tracked metadata (git branch, PR status, ports, notifications). Represented by `WorkspaceInfo` ([types.ts:57-76](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L57-L76)). |
| **Pane** | A rectangular region of a workspace produced by splitting; holds one or more surfaces (tabs) and tracks which surface is active. Represented by `PaneInfo` ([types.ts:86-90](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L86-L90)). |
| **Surface** | A single tab within a pane: a terminal, browser, markdown viewer, or diff viewer. Represented by `SurfaceRef` (persisted, in the split tree) and `SurfaceInfo` (runtime view); `SurfaceType` is `'terminal' \| 'browser' \| 'markdown' \| 'diff'` ([types.ts:12-30](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L12-L30), [types.ts:79-83](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L79-L83)). |
| **Split Tree / SplitNode** | The immutable binary tree that lays out a workspace's panes. Each node is either a `leaf` (one pane, its surfaces, and the active surface index) or a `branch` (a split direction, a size ratio, and two child nodes) ([types.ts:8-10](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L8-L10)). Mutations always produce new tree objects via helpers such as `splitNode()`, `removeLeaf()`, and `findLeaf()` ([CLAUDE.md:149-151](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/CLAUDE.md#L149-L151)). |
| **Keep-Alive Tabs** | Design decision under which every terminal tab in a pane is rendered simultaneously (hidden ones use `visibility: hidden`); switching tabs only changes CSS, so the xterm.js instance and its PTY connection stay alive without reattachment ([CLAUDE.md:144-147](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/CLAUDE.md#L144-L147)). |
| **PTY** | A pseudo-terminal process (ConPTY on Windows) that backs a terminal surface, managed by `PtyManager`; keyed by `SurfaceId` so the PTY ID equals the surface ID for reliable re-attachment ([pty-manager.ts:192-193](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/main/pty-manager.ts#L192-L193), [CLAUDE.md:144-147](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/CLAUDE.md#L144-L147)). |
| **Branded IDs** | String types narrowed with a template-literal prefix so IDs of different kinds are not interchangeable at compile time: `WorkspaceId` (`ws-{uuid}`), `PaneId` (`pane-{uuid}`), `SurfaceId` (`surf-{uuid}`), `WindowId` (`win-{uuid}`), and `AgentId` (`agent-{uuid}`) ([types.ts:2-5](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L2-L5), [types.ts:128](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L128)). The uuid portion comes from `crypto.randomUUID()` ([id.ts:12-14](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/id.ts#L12-L14)). |
| **Named Pipe (V1/V2)** | The Windows named pipe `\\.\pipe\pandamux` that serves as the control plane. V1 is a legacy line-based text protocol (`V1Command`: command, surfaceId, args) used by shell integration hooks; V2 is an authenticated JSON-RPC protocol (`V2Request`/`V2Response`) used by the CLI and agents, with a small allowlist of methods (`system.identify`, `system.capabilities`, `hook.event`, `agent.activity`) exempt from the per-instance auth token ([pipe-server.ts:5-34](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/main/pipe-server.ts#L5-L34), [pipe-server.ts:62-72](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/main/pipe-server.ts#L62-L72)). |
| **Agent** | A spawned CLI process (e.g. an AI coding agent) attached to a pane/surface via `AgentManager.spawn()`, tracked with a lifecycle status of `'spawning' \| 'running' \| 'exited'` and identified by an `AgentId` (`agent-{uuid}`) ([types.ts:128-141](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L128-L141), [agent-manager.ts:24-34](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/main/agent-manager.ts#L24-L34)). |
| **CDP** | Chrome DevTools Protocol; used by `CDPBridge` to drive the browser panel (navigation, accessibility-tree snapshots, click/type/fill, screenshots, JS eval) over a webContents debugger session, returning a `CDPSnapshot` (accessibility tree plus a ref count) ([cdp-bridge.ts:1-15](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/main/cdp-bridge.ts#L1-L15), [types.ts:159-162](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L159-L162)). |
| **OSC Notification** | A terminal notification raised from inside a PTY via an ANSI OSC escape sequence rather than a pandamux CLI call. The xterm.js parser registers handlers for OSC 9 (iTerm2-style), OSC 99 (kitty-style key=value), and OSC 777 (rxvt-unicode `notify;title;body`), each translated into a `window.pandamux.notification.fire()` call; a plain terminal bell (`\x07`) is a throttled fallback for agents that only ring the bell ([useTerminal.ts:476-513](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/renderer/hooks/useTerminal.ts#L476-L513)). |
| **Shell Integration** | Per-shell hook scripts (bash/zsh, PowerShell, cmd) sourced in spawned terminals that report cwd, git branch/dirty state, shell state, and (bash/PowerShell) ports and PR status back to pandamux; they expose a `pandamux` shell function that shells out to the CLI ([pandamux-bash-integration.sh:1-9](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shell-integration/pandamux-bash-integration.sh#L1-L9)). |
| **Orchestrator Plugin** | `pandamux-orchestrator`, a bundled Claude Code plugin that decomposes a large task into subtasks, assigns each to a separate Claude Code instance, and runs them in parallel across PandaMUX panes organized into dependency-aware waves, ending with an automated cross-agent reviewer ([README.md:1-7](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/resources/pandamux-orchestrator/README.md#L1-L7)). Its runtime state shape is mirrored in the main process as `OrchestrationState` ([types.ts:372-386](https://github.com/BoardPandas/Pandamux/blob/0ab9e6463a9017a7b8ea98f10b3f847507658ac4/src/shared/types.ts#L372-L386)). |
